1. 개념 정의
클라우드 컴퓨팅이란, 물리적인 저장 장치나 서버를 직접 갖추지 않아도, 외부의 클라우드 서버를 통해 데이터를 저장하고 불러올 수 있는 기술입니다.
사용자는 컴퓨터를 24시간 켜 두지 않아도, 서버를 빌리면 안정적인 네트워크 서비스나 애플리케이션을 운영할 수 있습니다.
기존에는 기업이 직접 서버를 구매하고 관리해야 했지만, 클라우드를 이용하면 필요한 시점에 자원을 쉽게 늘리거나 줄일 수 있습니다.

2. 가상화
클라우드 컴퓨팅은 가상화를 통해 같은 컴퓨터 내에 서로 다른 1개 이상의 가상 컴퓨터를 운용할 수 있습니다. 
(즉, 서로 다른 운영체제를 같은 컴퓨터 안에서 실행할 수 있습니다.)
이를 통해 하드웨어 한 대를 여러 개의 독립된 가상 컴퓨터처럼 사용할 수 있게 되어, 서버의 자원이 불필요하게 낭비되지 않고 효율적으로 사용될 수 있게 됩니다.

3. 종류
클라우드 컴퓨팅은 클라우드에서의 관리 범위에 따라 세 종류로 나뉩니다. 
첫째, Iaas(Infrastructure as a Service)는 서버와 저장소 같은 인프라를 빌려 사용하는 형태입니다.
둘째, Paas(Platform as a Service)는 개발 환경 자체를 제공하는 서비스입니다. 개발자는 서버 설정이나 OS 설치를 직접 하지 않고도, 애플리케이션 개발에만 집중할 수 있습니다.
셋째, Saas(Software as a Service)는 완성된 소프트웨어를 인터넷을 통해 제공하는 서비스입니다. 
IaaS로 갈수록 사용자가 관리할 영역은 줄어들고, 클라우드가 관리하는 영역이 늘어납니다. 

4. 읽기/쓰기 속도
클라우드 저장소는 지역 데이터센터까지의 물리적 거리와 네트워크 혼잡도 때문에 사용자 컴퓨터의 로컬 저장장치(HDD, SDD)보다 느리다
이에 클라우드는 다음과 같은 방법으로 느린 속도를 보완합니다.
1) 자주 사용하거나 서버에서 실시간 계산이 필요한 데이터는 서버의 메모리 기반 DB에 저장해서 읽어오고 변경함
2) 이미지/영상 등 데이터가 크고 정적인 정보를 가지는 파일은 사용자와 가까운 서버인 CDN(Content Delivery Network)에서 제공
3) API 호출 시 필요한 데이터만 JSON 같은 가벼운 포맷으로 전송해 통신량을 최소화합니다.

5. 장점, 단점
클라우드의 가장 큰 장점은 비용과 유연성입니다. 필요한 만큼 자원을 구매하기 때문에 초기 장비 구입 비용이 들지 않고, 서비스 이용량이 늘어난 만큼 서버를 유연하게 확장할 수 있습니다. 또한 유지보수 비용 역시 클라우드 업체가 담당하기 때문에 관리 부담이 줄어듭니다.
반면 단점도 존재합니다.
외부 서버와의 연결이 끊기면 서비스를 이용할 수 없고, 기업이 직접 데이터를 관리하는 것이 아니기 때문에 보안 문제가 생길 수 있습니다.
